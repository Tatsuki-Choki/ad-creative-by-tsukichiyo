# Ad Creative

このディレクトリはMeta広告用クリエイティブをAI（nanobanana2 / Google Gemini API）で生成・管理するためのワークスペースです。

## 「スタート」と言われたら

以下の対話型ワークフローを開始してください。必ず日本語で応答すること。

### Phase 1: ヒアリング

ユーザーに以下を順番に質問する（まとめて聞いてOK）:

1. **クリエイティブの種類は？**
   - バナー（1200x628px / Meta フィード広告）
   - SNS投稿（1080x1080px / Meta フィード投稿）
   - ストーリーズ（1080x1920px / Meta ストーリーズ広告）
   - 複数選択可

2. **何を宣伝する？**（商品・サービス・キャンペーン内容）

3. **ターゲット層は？**（年齢・性別・興味関心など）

4. **トーン・雰囲気は？**（例: 高級感、カジュアル、信頼感、ポップ、ミニマルなど）

5. **入れたいテキスト・コピーはある？**（キャッチコピー、CTA、価格など）

6. **リファレンス画像はある？**
   - `references/` に画像があれば確認する
   - なければスキップ

7. **何枚生成する？**（デフォルト: 3枚）

### Phase 2: プロンプト生成

1. `brand-assets/guidelines.md` を読み込み、ブランドカラー・フォント・ロゴ情報を把握
2. `references/` にリファレンス画像があれば読み込んで分析
3. **`docs/` 内のプロンプトガイドを参照し、ベストプラクティスに従ってプロンプトを構築する:**
   - `docs/02_prompting-guide.md` — ゴールデンルール、ライティング・カメラ・雰囲気の指定方法
   - `docs/03_prompt-tips.md` — 5要素（スタイル/主題/場所/アクション/構図）、実践Tips、広告向けプロンプト例
   - `docs/01_api-reference.md` — 対応解像度・アスペクト比・APIパラメータ
4. ヒアリング内容 + ブランドガイドライン + リファレンス + ドキュメントのベストプラクティスを統合して、nanobanana2用のプロンプトを生成
5. プロンプトをユーザーに提示し、調整があれば反映
6. 確定したプロンプトを `prompts/{種類}/` にYAML形式で保存（ファイル名: `YYYYMMDD_概要.yaml`）
   - メタ情報（campaign, date, type, size, aspect_ratio, platform, target, tone）をYAMLヘッダに記載
   - 各プロンプトは `prompts` リスト内に name, prompt, output を持つ構造で保存

### Phase 3: 画像生成

1. `generate.py` を使って画像を生成:
   ```bash
   python generate.py --type {banner|sns|story} --prompt "プロンプト" --count {枚数}
   ```
2. 生成された画像を `{種類}/drafts/` から読み込んでユーザーに表示
3. 各画像について「採用 / 再生成 / 修正」を確認

### Phase 4: レビュー

1. ユーザーが「採用」とした画像を `{種類}/review/` に移動
2. 修正が必要な場合はプロンプトを調整して再生成
3. 全ての画像が決まったら、レビュー用の一覧を表示

### Phase 5: 承認・納品

1. ユーザーに最終確認を取る
2. 承認された画像を `{種類}/approved/` に移動
3. 完了サマリーを表示:
   - 承認済み画像の一覧（ファイル名・サイズ・用途）
   - 使用したプロンプト（保存先パス）
   - Meta広告マネージャへのアップロード手順

## フォルダ構成

```
ad-creative/
├── .env                  # GEMINI_API_KEY
├── generate.py           # nanobanana2 API呼び出し
├── docs/                 # nanobanana2公式ドキュメント（日本語）
│   ├── 01_api-reference.md    # API仕様・解像度・パラメータ
│   ├── 02_prompting-guide.md  # プロンプトガイド（実践編）
│   └── 03_prompt-tips.md      # プロンプトTips・広告向け例
├── brand-assets/         # ブランド素材（ロゴ・カラー・フォント）
├── references/           # リファレンス画像
├── prompts/              # プロンプトテンプレート
│   ├── banners/
│   ├── sns-posts/
│   └── stories/
├── banners/              # バナー広告
│   ├── drafts/           # 生成物
│   ├── review/           # レビュー中
│   └── approved/         # 承認済み
├── sns-posts/            # SNS投稿画像
│   ├── drafts/
│   ├── review/
│   └── approved/
└── stories/              # ストーリーズ広告
    ├── drafts/
    ├── review/
    └── approved/
```

## Meta広告サイズ規定

| 配置 | 推奨サイズ | アスペクト比 |
|------|-----------|-------------|
| フィード | 1080x1080px | 1:1 |
| フィード（横長） | 1200x628px | 1.91:1 |
| ストーリーズ/リール | 1080x1920px | 9:16 |

## 注意事項

- 画像生成にはGemini APIキー（`.env`）が必要
- `brand-assets/guidelines.md` のブランド情報は常に参照すること
- 生成画像のファイル名にはタイムスタンプが自動付与される
- ワークフローの途中で「やり直し」と言われたら該当フェーズに戻る
